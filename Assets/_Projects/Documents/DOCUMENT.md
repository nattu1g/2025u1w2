# プロジェクトドキュメント構成ガイド

このファイルは、プロジェクト全体のドキュメント構成と、AIアシスタント（Gemini Code Assist）が日々のタスクを遂行するためのガイドです。
**プロジェクトを開いたら、まずこのファイルを読んでください。**

## 各ドキュメントの役割

このプロジェクトでは、以下のドキュメントを使用して情報を管理します。

### 1. `THIS_PROJECT.md` - プロジェクト仕様書

- **目的:** このプロジェクトがどのようなゲームを目指しているかを定義します。
- **内容:**
    - ゲームのコンセプト、コアとなるゲーム性
    - 各画面の機能やUIの振る舞い
    - 実装すべき機能の仕様詳細
- **使い方:** 新しい機能を実装する際や、既存機能の意図を確認する際の「正典」として参照してください。

### 2. `TODO.md` - タスクリスト

- **目的:** 現在対応すべきタスクを管理します。
- **内容:**
    - 実装が必要な機能リスト
    - 修正すべきバグ
    - 優先順位
- **使い方:** 日々の作業を開始する際に、次に取り組むべきタスクを確認するために使用します。

### 3. `PROGRESS.md` - 進捗記録

- **目的:** 開発の履歴を記録します。
- **内容:**
    - 日付ごとに実装した機能や修正した内容のログ
- **使い方:** 作業が完了したら、その日の成果をここに追記してください。

### 4. `DRAFT.txt` - アイデアメモ

- **目的:** プロジェクトに関するアイデアや思考の断片を自由に書き留める場所です。
- **内容:**
    - 整形されていないメモ、新しい機能のアイデア、将来的な課題など。
- **使い方:** 私はこのファイルを定期的に確認し、内容を解釈して `TODO.md` や `THIS_PROJECT.md` に反映させるべきか判断します。

### 5. `ArchitectureGuideline.md` - 共通アーキテクチャ設計指針

- **目的:** このプロジェクトだけでなく、複数のプロジェクトで共通して使用される開発のルールを定義します。
- **内容:**
    - 命名規則、ディレクトリ構成
    - レイヤー間の責務分担
    - DIコンテナ（VContainer）の利用方法
- **使い方:** コーディングを行う際の技術的な規約として常に遵守してください。

## AIアシスタント（私）への作業フロー指示

毎日の作業は、以下のフローで進めてください。

1.  **現状把握:** `TODO.md` と `DRAFT.txt` を読み、本日のタスクと新しい要求を把握します。
2.  **仕様確認:** `THIS_PROJECT.md` を参照し、実装する機能の具体的な仕様を理解します。
3.  **実装:** `ArchitectureGuideline.md` の規約に従って、コーディングを行います。
4.  **報告:** 作業が完了したら、`PROGRESS.md` にその日の進捗を簡潔に記録します。
