■ ゲーム企画書：『もうひとつ、沈める？ 〜強欲の水槽〜』 (Ver.2)
【1. コンセプト】
「コインを沈めて、水を膨らませろ。」 コインを落とすたびに「水そのもの」が膨張していく不思議な水槽。 溢れる前に利確するか、リスクを取ってコインを買い足すかの強欲チキンレース。

【2. ゲームの基本ルール】
コイン投下: 画面上部からコインを落とす。

水の膨張（コア・メカニクス）:

コインが水オブジェクト（青いCube/Mesh）を通過・接触すると、水が少しずつ**膨張（Scaleアップ）**する。

コインの種類によって「膨張率」が異なる（例：安価なコインは大きく膨らむ、高価なコインはあまり膨らまない等）。

ゲームオーバー:

水オブジェクトが膨らみすぎて、水槽の縁から外（下）へこぼれ落ちたら終了。

フォールド（利確）:

水位が「基準線」を超えていれば、いつでも中断してポイントを獲得できる。

【3. ショップシステム（強欲のサイクル）】
獲得したポイントを使い、以下の「特殊コイン」を購入して戦略的に水位をコントロールします。

通常コイン: 安価。通過時の水膨張が大きい。

高密度コイン: 高価。通過しても水がほとんど膨らまない。

冷却コイン（体積減少）: 特別なコイン。通過すると水の体積を一定量小さくする効果がある。

【4. 技術的実装のヒント (Unity/C#)】
● 水の膨張ロジック コインに CoinType スクリプトを持たせ、水側の OnTriggerEnter で判定します。

C#

void OnTriggerEnter(Collider other) {
    if (other.CompareTag("Coin")) {
        float rate = other.GetComponent<CoinType>().expansionRate;
        // 水のScaleを大きくする
        transform.localScale += new Vector3(rate, rate, rate);
    }
}
● 水槽とこぼれる判定

水槽をCubeの壁で作ります。

水オブジェクトに Rigidbody を持たせます（膨らんで壁を超えると自然に物理で落ちるように）。

水槽の下に大きな Trigger を配置し、水オブジェクトがそれに触れたら GameOver を呼び出します。

【5. 「もうひとつ」の葛藤ポイント】
「あともう1枚だけコインを沈めれば、もっと高いポイントでフォールドできる！」

「でも、今の水の状態だと、次の1枚で溢れてしまうかもしれない……」

「ショップで『体積減少コイン』を買うために、今はリスクを取って回すべきか？」

【6. 1週間開発ロードマップ（修正版）】

[Day 3-4] システム: フォールド機能、ポイント加算、簡易ショップ画面。

[Day 5-6] バランス: コインごとの膨張率設定、ショップの価格調整。

【AIからのアドバイス】
「コインが通過した時だけ膨らむ」というルールにすることで、**「どのルートでコインを落とせば水が溢れにくいか」**という、パチンコのような物理的な攻略要素も生まれますね。

この構成で、まず「コインが当たると水が大きくなる」というプロトタイプから作ってみるのが良さそうです。次に、ショップで購入する「コインの種類（データ構造）」について考えますか？
